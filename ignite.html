Give your founders an AI cofounder on day one

Fork Ignite, add your platform-specific skills, and ship a customized agent to every team that launches through you. Your founders get Meteora's ecosystem intelligence plus your platform's unique value.

1 Fork the repo

gh repo fork MetClaw/meteora-ignite --clone

Start with all 8 base skills. Add your own on top.

2 Add your skills

mkdir skills/your-platform-onboard

Create platform-specific skills: onboarding flows, custom pool configs, partner integrations.

3 Ship to your founders

git clone https://github.com/you/meteora-ignite ~/.claude/skills/ignite

One install command for every founder in your program.

4 Contribute back

gh pr create --title "Add [Platform] skills"

Your best skills can become part of the base layer for everyone.

Partner examples: EasyA could add hackathon-to-launch pipeline skills. Bags could add bonding curve optimization. Any launchpad can plug in their unique flow and give founders a smarter experience.

Build on Ignite's knowledge layer

Ignite skills are plain markdown files. No framework, no SDK, no lock-in. Works with any LLM that supports system prompts or tool use. Build agents, bots, dashboards, or entirely new products on top.

1 Use as CLI skills

~/.claude/skills/ignite/skills/*/SKILL.md

Each skill is a self-contained SKILL.md file. Drop into any Claude Code compatible agent.

2 Use as system prompts

cat skills/ignite-liquidity/SKILL.md | your-llm-api

Feed any skill directly to GPT-4, Claude, Gemini, or local models as a system prompt.

3 Build a Telegram bot

npm install telegraf @anthropic-ai/sdk

Wire Ignite skills into a Telegram bot. The always-on cofounder that lives in your group chat.

4 Build an MCP server

npm install @modelcontextprotocol/sdk

Expose Ignite capabilities as MCP tools. Any MCP-compatible client gets Meteora intelligence.

Architecture is intentionally simple. Skills are markdown. No dependencies, no build step, no runtime. Fork it, extend it, ship it. The knowledge is the product.
